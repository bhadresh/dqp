Hello, Im developing an application that loads a remote kml file
generated by a PHP script. For the communication between client and
server side Im using a KML Network Link that I attach to the globe
while loading the map instance. From that point on, the plugin manages
the request and retrieval of data automatically. The code looks like
this:

var kmlActualizacion="<?xml version='1.0' encoding='UTF-8'?><kml
xmlns='http://www.opengis.net/kml/2.2'><NetworkLink>
<name>Inmuebles</name><open>1</open><Link><href>"+urlServ+"?</href><viewFor mat>BBOX=[bboxWest],[bboxSouth],[bboxEast],[bboxNorth],[cameraAlt]</viewFor mat><httpQuery>sesion="+idSesion+"</httpQuery><viewRefreshMode>onStop</view RefreshMode><viewRefreshTime>0.0</viewRefreshTime></Link></NetworkLink></km l>";

var kmlObject = ge.parseKml(kmlActualizacion);
ge.getFeatures().appendChild(kmlObject);

The server generates KML files containing placemarks, and each one of
them contains HTML and Javascript code embedded inside the description
tag. Here is a sample fragment:

<?xml version="1.0" encoding="UTF-8"?><kml
xmlns="http://www.opengis.net/kml/2.2"><Document><Placemark><name>hola</name><description><![CDATA[<a
href="#" onclick='alert("hola");'> Click to
see</a><br>]]></description><Point><coordinates>-3.7462063,40.3754728999999 98</coordinates></Point></Placemark></Document></kml>

When the globe retrieves this data via the network link, it renders the
placemark and the ballon well, but when i click the link inside the
balloon, the alert just wont show up. I did a DOM inspection and found
out that the onclick attribute is missing, it only remains the href
one. Im not quite sure when exactly is this info ripped off the kml. Im
guessing the Network Link is doing some kind of transformation that i
cant figure out, checked out the KML documentation to see wether theres
any specifications about the behaviour in regards to this but didnt get
any clue. So im turning to you just in case anybody knows whats wrong,
cause this has been bugging me for some time now. Thanks in advance.

Hi,

Encapsulating the functionality in Kml as much as possible is a
*great* idea...it means you can share your rich GIS widely!

However, the issue here is a security one and must be dealt with at
the client end.

I think that the change would be fairly trivial though, simply place
the following function in the client code.
It will catch all clicks onto placemarks and allow the script to run.

// catch all placemark clicks and call getBalloonHtmlUnsafe
google.earth.addEventListener(ge.getWindow(), 'mousedown',
function(event) {
  if (event.getTarget().getType() == 'KmlPlacemark' ') {
  event.preventDefault();
  var placemark = event.getTarget();
  var content = placemark.getBalloonHtmlUnsafe();
  var balloon = ge.createFeatureBalloon('');
  balloon.setFeature(placemark);
  balloon.setContentString(content);
  ge.setBalloon(balloon);
  }

});

Hope that helps.

F.
